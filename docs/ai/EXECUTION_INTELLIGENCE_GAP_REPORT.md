# Execution Intelligence Gap Report

日期: 2026-05-25

## 发现的问题

1. 截图中的两个“执行中”任务并不是真的 AI 正在执行。数据库里它们没有 `TaskRun`,没有助手消息,没有工具调用,只是 `task.status = doing`。
2. “查天气”任务被产品经理执行后,只调用了 `current_datetime`,没有查天气。原因是该助手没有天气工具、没有 `fetch_url`,也没有 `run_command`;系统也没有根据任务意图换到更合适的助手。
3. 当前 UI 容易把普通任务状态和真实执行运行时混在一起。用户看到“执行中”会以为 AI 正在跑,但真实依据应该是 `TaskRun.status`。
4. 执行结果目前散在 DM、Activity、TaskRun.output,缺少一个任务级“执行详情/报告”入口。

## 已由 Codex 清理的数据

以下两个任务因没有 `TaskRun`,已从 `doing` 重置为 `todo`,并写入审计说明:

- `【技术】iframe 沙箱 PoC + AI 生成互动绘本可行性验证 — 不依赖其他角色，技术侧可立即启动`
- `【教研】基于 PRD 的 5 步任务结构，设计每一步的教学目标、引导话术、system prompt 框架 — 教研架构师输出`

## 期望修复方向

- 只有真实 `TaskRun` 才能显示 AI 执行状态。
- 给所有 AI 指派任务提供明确“开始执行”入口。
- 增加工具/Agent 路由:任务需要联网、终端、代码能力时,不要交给无对应 skill 的助手空答。
- 天气类任务应先补齐城市,再用真实工具查询;不能只回复“没有城市/没有工具”就结束。
- 增加执行详情/报告面板,集中展示 AI 回复、工具调用、审批记录、最终结果。
